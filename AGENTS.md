## 关于 CDROID
- CDROID 是一个基于 CAIRO 的高仿 Android GUI 框架，目标是把 Android 官方 SDK 中的 Java 组件逐行移植到 C++。
- 所有组件、成员变量、接口签名、行为语义都以 Android 官方实现为准。
- 每个函数都尽量与 Android 原逻辑保持一致，优先逐行对应，而不是凭空造轮子。

## 工作原则
- 先看 Android 官方源码或现有实现，再动手修改；遇到不确定点优先查参考实现，不凭猜测。
- 先定位根因，再做最小修复；遇到问题时先复现、再分析、再修改。
- 修改前先确认影响范围，尽量保持现有行为和兼容性不变。
- 如有必要可补充日志，但不要把调试痕迹留在正式逻辑里；需要时应及时清理。
- 任何变更都要以“可验证”为目标：修改后要重新编译，必要时做回归验证。

## 代码迁移要求
- 组件移植要尽量逐行对应 Android Java 逻辑，不要随意重构。
- 保持与 Android 官方 SDK 一致的成员变量名、方法签名和调用关系。
- 对于 UI、绘制、颜色、动画、输入等行为，优先实现 Android 原生语义。
- 兼容性优先，全面兼容 Android 是最高准则。
- 代码里面注释用英文 

## 构建与验证
- 构建目录：outX64-Debug 或 outX64-Release
- 构建命令：cd outX64-Debug && make cdroid -j44
- 完成任何修改后，必须重新执行构建验证；只有拿到明确构建结果，才能认为任务完成。
